Common Questions
Q: Does anything get installed on the suspect's machine?
No. The collector.py runs directly. The .exe version requires zero installation. Both leave no permanent traces beyond the output JSON file.
Q: Does the tool decrypt browser sync data?
No. X-SyncTrace only reads metadata — timestamps, device IDs, sync states, and LevelDB keys. Encrypted content is never accessed. This keeps the tool within legal and ethical forensic boundaries.
Q: Is the evidence admissible in court?
Evidence collected using X-SyncTrace is admissible when: (1) proper acquisition procedure is followed, (2) SHA-256 hash integrity is verified, (3) chain of custody is documented, and (4) the examiner can explain the methodology. The tool is compliant with ISO/IEC 27037 and NIST SP 800-86.
Q: Why does the Lovable app redirect to the browser instead of collecting files?
Browser-based apps cannot access the local filesystem due to browser sandboxing (security policy). X-SyncTrace solves this with the two-component architecture: Python collector (runs on machine with filesystem access) + this HTML dashboard (analyzes the exported JSON data).
